# spring-board-api

백엔드 신입 웹개발자 취업용 포트폴리오를 목표로 하는 게시판 API 프로젝트입니다.

이 프로젝트는 단순히 기능만 많이 넣는 방향보다, 게시판 백엔드의 기본기를 충실히 구현하고 그 설계 이유를 설명할 수 있도록 만드는 것을 목표로 합니다. 기능 구현과 함께 요구사항, 설계 결정, 트러블슈팅을 문서화하여 GitHub에서도 개발 과정이 드러나도록 관리합니다.

## 프로젝트 목표

- Spring Boot 기반 게시판 백엔드 API 구현
- 실무에서 자주 사용하는 계층 구조와 예외 처리, 검증, 인증/인가 흐름 학습
- 기술 면접에서 설계 의도와 구현 포인트를 설명할 수 있는 포트폴리오 완성
- 기능 추가 과정과 의사결정을 문서로 남기는 개발 습관 정착

## 개발 원칙

- 단계별로 범위를 나누어 개발한다.
- 기본기가 드러나는 구조를 우선한다.
- 과도한 기술 도입보다 설명 가능한 수준의 설계를 선택한다.
- 기능 구현과 문서화를 함께 진행한다.

## 기술 스택

- Java 17
- Spring Boot 3.x
- Spring Security
- JWT
- Spring Data JPA
- QueryDSL
- MySQL
- SpringDoc OpenAPI (Swagger)
- JUnit5 / Mockito

## 개발 단계

### 1차 MVP

핵심 게시판 API와 인증/인가의 기본 흐름을 안정적으로 완성하는 단계입니다.

- Spring Boot 프로젝트 생성 및 환경 설정
- DB 연결 및 JPA 설정
- 공통 응답 형식 `ApiResponse<T>`
- 전역 예외 처리
- Swagger 설정
- JPA Auditing 설정
- 회원가입 / 로그인 / 내 정보 조회
- 게시글 작성 / 목록 조회 / 상세 조회 / 수정 / 삭제
- 댓글 작성 / 조회 / 수정 / 삭제
- 게시글 페이지네이션 및 검색
- 작성자 본인 여부 기반 권한 처리
- 기본 서비스 단위 테스트

### 2차 확장

MVP를 안정화한 뒤 인증 체계와 관리 기능을 확장하는 단계입니다.

- Refresh Token 발급 및 재발급
- 로그아웃과 Refresh Token 무효화
- 카테고리 관리
- 관리자 권한 기능
- 대댓글(1 depth)
- QueryDSL 기반 동적 검색 고도화

### 3차 고도화

포트폴리오 완성도를 높이기 위한 운영/확장 기능 단계입니다.

- 파일 업로드 및 다운로드
- 로컬 스토리지 또는 S3 연동
- 테스트 범위 확장
- 배포 환경 정리
- 성능 및 운영 관점 개선

## 진행 방식

이 프로젝트는 아래 흐름으로 진행합니다.

1. 문서로 범위를 확정한다.
2. Spring Boot 프로젝트를 생성한다.
3. 공통 설정과 도메인 기반 구조를 먼저 만든다.
4. 회원, 게시글, 댓글 기능을 단계적으로 구현한다.
5. 각 단계마다 문서를 갱신하고 커밋으로 기록을 남긴다.
