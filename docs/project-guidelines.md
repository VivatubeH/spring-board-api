# 프로젝트 운영 지침

## 문서 목적

이 문서는 `spring-board-api` 프로젝트를 어떤 기준과 루틴으로 진행할지 정리한 운영 지침입니다.

이 프로젝트의 목표는 단순히 기능이 많은 게시판을 만드는 것이 아니라, 백엔드 신입 개발자로서 기본기를 보여주고 기술 면접에서 설계 의도와 구현 이유를 설명할 수 있는 포트폴리오를 완성하는 것입니다.

프로젝트를 진행하는 동안 방향이 흔들리지 않도록, 개발 원칙과 문서화 기준, 일일 진행 루틴을 이 문서를 기준으로 관리합니다.

## 프로젝트 방향

- 이 프로젝트는 백엔드 신입 웹개발자 취업용 게시판 포트폴리오를 목표로 한다.
- Spring Boot 기반 게시판 백엔드 프로젝트를 중심으로 구현하되, Thymeleaf를 사용해 핵심 화면까지 함께 구성한다.
- 기능을 과하게 넓히기보다, 기본기에 충실하고 설명 가능한 수준으로 완성하는 것을 우선한다.
- 결과물뿐 아니라 개발 과정과 의사결정이 문서와 커밋에 남도록 진행한다.

## 개발 원칙

- 항상 단계별로 진행한다.
- 구현 전에 현재 단계의 목표와 범위를 먼저 확인한다.
- 최종 목표와 실제 개발 범위를 구분하고, 실제 구현은 MVP부터 완성한다.
- 과도한 기술 도입보다 안정적인 완성과 이해 가능한 설계를 우선한다.
- 기능 구현과 문서화를 함께 진행한다.
- 나중에 면접에서 설명할 수 있는 선택인지 계속 점검한다.

## 문서화 원칙

- 기능 추가나 구조 변경이 생기면 관련 문서를 함께 갱신한다.
- 요구사항, 설계 결정, 예외 처리 방식, 트러블슈팅은 `docs/`에 기록한다.
- `README.md`는 프로젝트 소개와 전체 흐름을 보여주는 메인 문서로 관리한다.
- 내부 설계와 학습 기록은 `docs/` 문서에 누적한다.
- 커밋 기록만 봐도 개발 흐름이 보이도록 작은 단위로 나누어 작업한다.

## 구현 우선순위

### 1차 MVP

핵심 게시판 기능과 인증의 기본 흐름을 안정적으로 완성하는 단계다.

- 회원가입 / 로그인 / 내 정보 조회
- 게시글 CRUD
- 댓글 CRUD
- 공통 응답 형식
- 전역 예외 처리
- 입력값 검증
- Swagger
- JPA Auditing

### 2차 확장

MVP를 안정화한 뒤 인증 체계와 관리 기능을 넓히는 단계다.

- Refresh Token
- 로그아웃 / 토큰 재발급
- 카테고리 기능
- 관리자 기능
- 대댓글
- QueryDSL 기반 검색 고도화

### 3차 고도화

포트폴리오 완성도를 높이기 위한 운영 및 확장 단계다.

- 파일 업로드
- S3 연동
- 테스트 확대
- 배포 및 운영 환경 개선

## 화면 구성 원칙

- 이 프로젝트는 백엔드 중심으로 진행한다.
- 화면은 Thymeleaf 기반으로 최소 기능부터 단계적으로 붙인다.
- 초반에는 백엔드 구조, 공통 설정, 도메인 설계에 집중한다.
- 화면은 백엔드 흐름이 어느 정도 안정된 뒤 필요한 범위부터 연결한다.
- 프론트엔드 기술은 프로젝트 목적에 맞는 수준에서 선택하고, 백엔드 포트폴리오라는 중심을 유지한다.

## 최종 목표

이 프로젝트의 최종 목표는 다음과 같다.

- 게시판 백엔드의 핵심 기능을 직접 설계하고 구현한다.
- 인증/인가, 예외 처리, 검증, 데이터 접근, 테스트 등 백엔드 기본기를 보여준다.
- 화면까지 포함한 결과물을 통해 프로젝트 이해도를 높인다.
- GitHub 문서와 커밋 기록만 봐도 개발 과정이 자연스럽게 드러나도록 완성한다.
