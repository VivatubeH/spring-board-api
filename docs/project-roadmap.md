# spring-board-api 3주 로드맵

## 목표
- 3주 안에 게시판 포트폴리오 프로젝트를 MVP 기준으로 완성한다.
- 백엔드 기본기와 문서화 흐름이 보이도록 만든다.
- 실행, 시연, 설명이 가능한 상태에서 마무리한다.

## 완료 기준
- Spring Boot 프로젝트가 정상 실행된다.
- 회원, 게시글, 댓글 핵심 기능이 동작한다.
- 최소한의 Thymeleaf 화면이 있다.
- README 및 docs 문서가 정리되어 있다.
- 기술 면접에서 구현 이유와 흐름을 설명할 수 있다.

## 운영 원칙
- 하루 목표를 과하게 잡지 않는다.
- 구현과 문서화를 같이 진행한다.
- dev-log는 짧게, work-notes는 자세히 기록한다.
- 하루 작업이 끝나면 저녁 기본기 학습 주제를 연결한다.
- 3주차가 끝나면 욕심내서 확장하지 말고 쇼핑몰 프로젝트로 넘어간다.

## 비고
- Refresh Token, 관리자 기능, 파일 업로드, 대댓글, QueryDSL 고도화는 1차 완료 후 확장 항목으로 둔다.
- 이번 게시판 프로젝트는 “완벽함”보다 “완성도 있는 MVP”를 목표로 한다.
