# 프로젝트 범위 및 로드맵

## 프로젝트 개요

이 프로젝트는 신입 백엔드 개발자 포트폴리오를 위한 게시판 API 프로젝트입니다.  
핵심 목표는 게시판 서비스에서 자주 요구되는 기능을 직접 구현하고, 각 설계 선택의 이유를 설명할 수 있는 수준까지 이해하는 것입니다.

전체 기능을 한 번에 구현하지 않고, `1차 MVP → 2차 확장 → 3차 고도화` 순서로 진행합니다.

## 왜 단계별로 개발하는가

- 기능 범위를 통제하여 완성도를 높이기 위해
- 핵심 CRUD와 인증/인가의 기본기를 먼저 안정화하기 위해
- GitHub에 실제 개발 흐름이 보이도록 하기 위해
- 면접에서 "왜 이 기능을 나중 단계로 미뤘는지"까지 설명할 수 있도록 하기 위해

## 최종 목표 범위

### 프로젝트 기반 설정

- Spring Boot 프로젝트 구성
- DB 연결 및 JPA 설정
- 공통 응답 형식 구성
- 전역 예외 처리
- Swagger 설정
- CORS 설정
- JPA Auditing 설정

### 회원 기능

- 회원가입
- 이메일 중복 검사
- 로그인
- Access Token + Refresh Token 발급
- 토큰 재발급
- 로그아웃
- 내 정보 조회
- 닉네임 / 비밀번호 수정
- 회원 탈퇴

### 카테고리 기능

- 카테고리 생성 / 수정 / 삭제
- 카테고리 목록 조회

### 게시글 기능

- 게시글 작성 / 수정 / 삭제
- 게시글 단건 조회
- 게시글 목록 조회
- 게시글 검색
- 카테고리별 게시글 조회

### 댓글 기능

- 댓글 작성 / 수정 / 삭제
- 게시글별 댓글 목록 조회
- 대댓글 작성 / 조회

### 파일 첨부 기능

- 이미지 업로드
- 게시글 삭제 시 첨부파일 삭제
- 파일 다운로드

### 관리자 기능

- 회원 목록 조회
- 회원 강제 탈퇴
- 게시글 강제 삭제
- 댓글 강제 삭제

## 1차 MVP 범위

### 목표

회원 인증의 기본 흐름과 게시판 핵심 CRUD를 안정적으로 구현한다.

### 포함 기능

- Spring Boot 프로젝트 생성 및 기본 의존성 세팅
- DB 연결 및 JPA 설정
- 공통 응답 형식 `ApiResponse<T>`
- 전역 예외 처리
- Swagger 설정
- JPA Auditing 설정
- 회원가입
- 로그인
- 내 정보 조회
- 게시글 작성 / 조회 / 수정 / 삭제
- 게시글 목록 조회
- 게시글 페이지네이션
- 게시글 검색
- 댓글 작성 / 조회 / 수정 / 삭제
- 작성자 본인 기준 권한 제어
- 서비스 레이어 단위 테스트 기초 작성

### 제외 기능

- Refresh Token
- 로그아웃
- 토큰 재발급
- 카테고리 관리자 기능
- 대댓글
- 파일 업로드
- 관리자 전용 강제 삭제

### 제외 이유

- 인증과 게시판 핵심 흐름을 먼저 안정화하는 것이 우선이기 때문
- 범위를 줄여 완성도 높은 첫 버전을 만들기 위해
- 추후 확장 포인트를 문서와 커밋으로 명확히 남기기 위해

## 2차 확장 범위

### 목표

인증 체계를 보강하고 관리 기능과 검색 구조를 확장한다.

### 포함 기능

- Refresh Token 발급 및 저장 전략 적용
- Access Token 재발급
- 로그아웃
- 카테고리 CRUD
- 관리자 권한 기능
- 대댓글 1 depth
- QueryDSL 기반 동적 검색

## 3차 고도화 범위

### 목표

실서비스 관점에서 부가 기능과 운영 편의성을 높인다.

### 포함 기능

- 파일 업로드 / 다운로드
- 로컬 또는 S3 저장소 연동
- 테스트 커버리지 확대
- 배포 환경 구성
- 문서 및 트러블슈팅 정리 강화

## 면접에서 설명할 핵심 메시지

- 처음부터 모든 기능을 넣기보다 MVP를 먼저 완성하는 방식으로 범위를 관리했다.
- 인증/인가와 게시판 CRUD, 예외 처리, 검증 같은 백엔드 기본기를 우선 구현했다.
- 이후 확장 단계에서 Refresh Token, 관리자 기능, 파일 업로드 등을 추가하도록 설계했다.
- 문서화와 커밋 기록을 통해 개발 과정과 설계 의사결정을 남겼다.
