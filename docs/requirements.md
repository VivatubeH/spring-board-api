# 요구사항 정의서

## 1. 프로젝트 소개

- 프로젝트명: `spring-board-api`
- 프로젝트 유형: 게시판 백엔드 API
- 목적: 신입 백엔드 개발자 포트폴리오 제작

## 2. 핵심 목표

- 게시판 서비스의 핵심 기능을 백엔드 관점에서 직접 구현한다.
- 인증/인가, 예외 처리, 검증, 데이터 접근 등 기본기를 보여준다.
- 기술 면접에서 구조와 선택 이유를 설명할 수 있도록 문서화한다.

## 3. 사용자 역할

### 공통

- 비회원(Guest)
- 일반 회원(User)
- 관리자(Admin)

## 4. 1차 MVP 기능 요구사항

### 4.1 회원

- 비회원은 회원가입을 할 수 있다.
- 비회원은 로그인할 수 있다.
- 로그인한 회원은 자신의 정보를 조회할 수 있다.
- 회원은 본인의 닉네임 또는 비밀번호를 추후 수정 가능하도록 확장 여지를 둔다.

### 4.2 게시글

- 로그인한 회원은 게시글을 작성할 수 있다.
- 모든 사용자는 게시글 목록을 조회할 수 있다.
- 모든 사용자는 게시글 상세를 조회할 수 있다.
- 게시글 목록은 페이지네이션을 지원해야 한다.
- 게시글 목록은 검색 기능을 지원해야 한다.
- 게시글 작성자 본인은 자신의 게시글을 수정할 수 있다.
- 게시글 작성자 본인은 자신의 게시글을 삭제할 수 있다.

### 4.3 댓글

- 로그인한 회원은 댓글을 작성할 수 있다.
- 모든 사용자는 게시글별 댓글 목록을 조회할 수 있다.
- 댓글 작성자 본인은 자신의 댓글을 수정할 수 있다.
- 댓글 작성자 본인은 자신의 댓글을 삭제할 수 있다.

### 4.4 공통

- API 응답 형식은 일관된 구조를 가져야 한다.
- 입력값 검증이 적용되어야 한다.
- 예외 상황에 대해 일관된 에러 응답을 제공해야 한다.
- 생성일시와 수정일시는 자동으로 관리되어야 한다.
- API 문서 확인이 가능해야 한다.

## 5. 비기능 요구사항

- 계층형 구조를 유지한다.
- 도메인 책임이 드러나도록 설계한다.
- 예외 처리는 전역에서 일관성 있게 관리한다.
- 테스트 가능한 구조를 지향한다.
- 문서를 통해 요구사항과 설계 과정을 추적 가능해야 한다.

## 6. 추후 확장 요구사항

- Refresh Token 기반 재발급
- 로그아웃 처리
- 카테고리 기능
- 관리자 기능
- 대댓글 기능
- 파일 업로드 기능
- QueryDSL 기반 검색 고도화

## 7. 현재 결정 사항

- 최종 목표 범위는 넓게 가져가되, 1차 구현은 MVP 중심으로 제한한다.
- 인증은 Spring Security + JWT를 사용한다.
- 데이터베이스는 MySQL을 기준으로 설계한다.
- 문서화는 `README.md`와 `docs/` 폴더를 기준으로 누적 관리한다.
